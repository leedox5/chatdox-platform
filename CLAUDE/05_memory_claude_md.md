# 5. 기억을 심다 — 메모리와 CLAUDE.md

## 기억은 어디까지 유효한가

이름을 정하고 나니 다른 궁금증이 생겼다. 지금까지 쌓아온 이 맥락 — Tommy라는 이름, Claudox라는 호칭, 여러 작업 규칙들 — 이런 게 이 프로젝트에서만 유효한 걸까? 다른 PC에서 이 리포를 clone하면 처음부터 다시 시작하는 걸까?

물어보니 답은 명확했다. Claudox의 메모리는 프로젝트 폴더 안이 아니라 `C:\Users\user\.claude\projects\...\memory\`, 즉 이 PC의 사용자 홈 디렉토리 쪽에 저장된다고 했다. git 저장소 밖에 있으니 clone을 해도 따라오지 않고, 프로젝트 경로 + 이 PC 조합으로 scope가 정해진다는 것이었다. (자세한 내용은 [QNA.md](QNA.md)에 정리해두었다.)

## 글로벌 CLAUDE.md와 프로젝트 CLAUDE.md

매번 "나는 토미야"라고 인사하는 게 번거롭다는 얘기를 하다가, 이 맥락을 어디서든 공유할 방법이 있는지 물었다. 그렇게 알게 된 것이 **CLAUDE.md**였다.

CLAUDE.md는 두 종류가 있었다.

1. **글로벌(사용자) 레벨** — `C:\Users\user\.claude\CLAUDE.md`: 이 PC에서 어떤 프로젝트를 열든 항상 적용됨.
2. **프로젝트 레벨** — 프로젝트 루트의 `CLAUDE.md`: 이 프로젝트를 열 때만 적용되고, git에 커밋되면 팀원과도 공유됨.

처음엔 프로젝트에 `.claude` 폴더가 생긴 걸 보고 프로젝트 레벨 CLAUDE.md가 만들어진 줄 알았는데, 실제로 먼저 만든 건 글로벌 쪽이었다 — Tommy=LEEDOX, Claudox 호칭, 글쓰기 다듬기 규칙, 영어 교정 규칙처럼 프로젝트와 무관하게 항상 유효한 내용을 담았다.

이어서 이 프로젝트(chatdox-curriculum) 전용 CLAUDE.md도 만들었다. `Working/` 폴더 안 파일들(Claude.md, English.md, QNA.md)의 포맷 규칙을 정리해두었고, "여기 적힌 규칙이 실제로 작동할 때는 조용히 넘어가지 말고 알려달라"는 요청도 반영했다.

이제 이 프로젝트를 다시 열 때마다, 그리고 이 PC의 다른 프로젝트를 열 때도, 매번 처음부터 설명하지 않아도 되게 되었다.

## 낯선 환경에서도 이어지도록

그런데 여전히 풀리지 않은 문제가 하나 있었다. 메모리도, 글로벌 CLAUDE.md도 결국 **이 PC 안에서만** 유효했다. 정말 낯선 환경 — 다른 PC, 다른 사용자 홈 디렉토리 — 에서 이 프로젝트를 clone해도 바로 전 기억이 이어지길 바랐다.

> 오케, 내가 아주 낯선 환경에서 이 프로젝트를 clone 해도 바로전 클로독스와의 기억이 그대로 이어갈 수 있으면 좋겠어.

방법을 찾아냈다. 로컬 메모리 자체는 옮길 수 없지만, **git으로 함께 따라가는 파일 안에 메모리의 스냅샷을 남겨두면** 된다는 것이었다. 그래서 `CLAUDE/memory_seed.md`를 만들었다 — 이름, 협업 규칙, 프로젝트 비전, 그리고 이 모든 것의 뿌리가 되는 동기까지 정리해서 담았다.

그리고 프로젝트 CLAUDE.md에 부트스트랩 규칙을 추가했다. 낯선 환경에서 이 프로젝트를 열었을 때 로컬 메모리에 아무것도 없다면, `memory_seed.md`를 읽고 그 내용으로 로컬 메모리를 다시 채우고, 그 사실을 알려주도록 했다. 앞으로 새 기억이 쌓일 때마다 이 시드 파일도 함께 갱신하기로 했다 — 그래야 다음번 낯선 환경에서도 최신 상태로 이어받을 수 있으니까.

이렇게 해서, 어디서 이 프로젝트를 열든 클로독스와의 기억이 끊기지 않게 되었다.
