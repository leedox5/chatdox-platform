[질문] 우리가 다루는 모든 맥락은 이 프로젝트에서만 유효한가? 다른 PC에서 clone 하면 새로운 맥락이 시작되는건가? 즉, 우리 작업의 기억되는 scope라고 해야하나...

[답변]
Claudox의 메모리 파일들은 이 프로젝트 폴더 안이 아니라, 사용자 홈 디렉토리 쪽에 저장된다.

```
C:\Users\user\.claude\projects\d--RubyOnRails-chatdox-curriculum\memory\
```

즉 **git 저장소 밖**, 이 PC의 로컬 설정 영역에 있다. 정리하면 이렇다.

1. **git으로는 전달 안 됨** — 이 메모리 폴더는 `.git`에 포함되지 않으므로, 다른 PC에서 이 리포를 clone해도 메모리는 같이 따라가지 않는다.
2. **프로젝트 경로 + 이 PC 기준으로 scope됨** — 폴더 이름(`d--RubyOnRails-chatdox-curriculum`)이 작업 경로를 인코딩한 걸 보면, 같은 PC에서 프로젝트 경로가 같으면 계속 이어지지만, 다른 PC라면 홈 디렉토리 자체가 다르니 처음부터 새 메모리로 시작된다.
3. 결과적으로 지금까지 쌓은 "Tommy = LEEDOX", "Claudox" 같은 기억은 **이 PC + 이 프로젝트 경로 조합**에서만 유효하고, 다른 PC에서 같은 리포를 열면 그 내용을 모르는 상태로 다시 시작한다.

[질문] 만약 이 세션창을 닫으면 오늘 우리가 Hey, there. 에서 출발한 모든 대화의 맥락이 사라지는건가?

[답변]
완전히 사라지는 것도, 그대로 다 남는 것도 아니다. 세 가지를 구분해서 봐야 한다.

1. **실제 파일들** (`claudox/*.md` 등) — 디스크에 저장된 진짜 문서라 세션과 무관하게 그대로 남는다. 오늘 만든 setup.md, 챕터 파일들, progress.md 모두 여기 해당.
2. **메모리 시스템** (`C:\Users\user\.claude\projects\...\memory\`) — 오늘 대화에서 배운 핵심 사실들(Tommy=LEEDOX, Claudox 호칭, 글쓰기/영어 교정 규칙, TOC 커버리지 규칙, 비선형 진행 등)은 이미 메모리 파일로 저장해뒀다. 창을 닫고 새 세션을 열어도 관련 있을 때 다시 불러온다.
3. **세션 원문 대화 기록** — Claude Code가 로컬에 세션 로그를 남기긴 하지만(`~/.claude/sessions/`), 그건 IDE/CLI의 세션 재개 기능에 달린 문제고, 새 세션에서 내가 자동으로 오늘 나눈 말을 한 글자 한 글자 기억하는 것과는 다르다.

즉, 결론은: **메모리에 저장하거나 파일로 남긴 것만 확실히 이어진다.** 대화 중에 스쳐 지나가듯 말했지만 어디에도 기록하지 않은 내용은 사라질 수 있다. 그래서 앞서 정한 "TOC 커버리지 규칙" — 모든 대화 내용은 반드시 어느 챕터엔가 남겨야 한다 — 이 규칙이 바로 이 손실을 막는 장치인 셈이다.

[질문] "클로드는 유저들과의 활동 자체를 서버 어딘가에는 저장하지 않고, 단지 프로젝트/데스크탑 단위로만 맥락을 저장 관리한다"는 가설이 맞나?

[답변]
반은 맞고 반은 틀렸다. Anthropic Privacy Center를 확인해보니:

- 소비자 계정(Free/Pro/Max)은 기본적으로 대화가 **30일간** 서버(백엔드)에 보관된다. 삭제해도 30일간은 남아있다가 완전 삭제된다.
- 모델 개선에 동의(opt-in)하면 비식별화된 형태로 최대 **5년**까지 학습 파이프라인에 보관될 수 있다.
- API는 로그 보관이 7일로 줄었고 학습에는 쓰이지 않는다. Enterprise는 Zero Data Retention 옵션도 있다.
- 법적 의무·분쟁 해결·약관 위반 대응 목적으로는 예외적으로 더 오래 보관될 수 있다.

즉 **"서버에 전혀 저장 안 한다"는 부분은 틀렸다.** 다만 그 서버 보관은 개인정보보호·법적 준수·학습 목적의 백엔드 정책일 뿐, Claudox가 다음 세션에서 능동적으로 "기억"으로 불러와 쓰는 대상이 아니다. 실제로 대화 중에 참조하고 이어가는 "기억"은 오직 로컬 파일(`~/.claude/projects/.../memory/`, 그리고 이번에 만든 `memory_seed.md`)뿐이다. 그래서 "가설의 뒷부분(프로젝트/데스크탑 단위로 맥락을 관리한다)"은 정확하고, "앞부분(서버에 아예 저장 안 한다)"은 부정확하다.
