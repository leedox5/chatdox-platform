# CodeRabbit PR 리뷰 TIPS

> CodeRabbit은 GitHub Pull Request(PR)의 변경 내용을 분석해 버그 가능성, 보안·성능 위험, 테스트 누락, 유지보수성 문제를 댓글로 알려주는 AI 코드 리뷰 도구입니다.

## 1. CodeRabbit이 하는 일

- PR 전체 요약과 변경 파일별 설명을 생성합니다.
- 코드 diff를 바탕으로 오류 가능성과 개선 제안을 인라인 댓글로 남깁니다.
- 프로젝트 규칙을 설정 파일과 경로별 지침으로 적용할 수 있습니다.
- 댓글을 바탕으로 수정 방향을 제안하거나, 활성화된 경우 자동 수정(Autofix)을 수행할 수 있습니다.
- 자동 리뷰를 기다리지 않고 PR 댓글에 `@coderabbitai review`를 작성해 수동 리뷰를 요청할 수 있습니다.

CodeRabbit은 보조 리뷰어이지 최종 승인자가 아닙니다. 모든 댓글을 그대로 적용하지 말고, 실제 요구사항·테스트·보안 정책과 대조해 판단합니다.

## 2. PR에서 자주 보는 리뷰 유형

| 유형 | 의미 | 대응 |
|---|---|---|
| Bug / Error | 실제 오동작 가능성 | 재현 조건을 확인하고 우선 수정 |
| Security | 인증·인가·비밀정보·입력 검증 위험 | 높은 우선순위로 검토 |
| Performance | 불필요한 쿼리·반복 호출·병목 | 데이터 규모와 운영 환경을 기준으로 판단 |
| Test | 회귀 테스트 또는 경계값 테스트 부족 | 테스트 추가 후 재검토 |
| Improvement | 가독성·구조·유지보수성 제안 | 현재 범위와 비용을 보고 선택 |
| Nitpick | 스타일·표현 선호 차이 | 프로젝트 규칙에 맞을 때만 반영 |

## 3. 리뷰 대응 순서

1. 먼저 사람이 핵심 변경 경로와 요구사항을 직접 확인합니다.
2. CodeRabbit 댓글을 `수정 필요`, `확인 후 보류`, `오탐/미적용`으로 분류합니다.
3. 버그·보안·데이터 손상 가능성이 있는 항목부터 수정합니다.
4. 수정 후 관련 테스트와 린트를 실행합니다.
5. 커밋을 추가하고 CodeRabbit의 재리뷰를 기다리거나 `@coderabbitai review`를 호출합니다.
6. 적용하지 않은 댓글에는 이유를 짧게 답변해 리뷰 기록을 남깁니다.

## 4. 좋은 답변 예시

```text
수정했습니다. 결제 금액은 서버의 상품 가격을 기준으로 검증하고,
회귀 방지를 위해 금액 불일치 테스트를 추가했습니다.
```

```text
확인 결과 현재 범위에서는 의도된 동작입니다.
이 엔드포인트는 관리자 전용 라우트이며 Authorization 정책 테스트가 이미 있습니다.
```

```text
이번 PR의 범위를 벗어난 개선 제안으로 판단해 별도 이슈로 기록하겠습니다.
```

## 5. 오탐을 줄이는 프로젝트 설정

저장소의 `.coderabbit.yaml` 또는 CodeRabbit 설정에서 다음을 명확히 합니다.

- 프로젝트의 빌드·테스트·린트 명령
- 보안 및 개인정보 처리 규칙
- 결제·인증 등 민감한 경로에 대한 별도 지침
- 생성 파일, vendor, fixture 등 리뷰 제외 경로
- PR 크기와 리뷰 우선순위

경로별 지침을 사용하면 예를 들어 `app/services/payments/**`에는 서버 금액 검증과 멱등성 확인을 요구하고, 문서 파일에는 링크·코드 예시의 일관성을 검사하도록 분리할 수 있습니다.

## 6. 이 프로젝트에서의 체크 포인트

- Toss Payments/PortOne 결제 상태는 브라우저 응답만 믿지 않고 서버 API로 재조회하는가?
- 결제 금액·통화·주문 ID가 서버 값과 일치하는가?
- `provider`별 기존 구독 갱신이 올바른 게이트웨이를 사용하는가?
- API Secret, Webhook Secret, billing key가 로그와 브라우저에 노출되지 않는가?
- 웹훅과 콜백이 중복 수신되어도 멱등적으로 처리되는가?
- 인증·인가 변경에 대한 실패 케이스 테스트가 있는가?

## 7. 주의사항

- AI 리뷰는 코드 실행, 운영 데이터, 비즈니스 의도를 완전히 이해하지 못할 수 있습니다.
- 제안된 수정 코드를 바로 붙여넣지 말고 diff와 테스트 결과를 확인합니다.
- 비밀키, 고객 개인정보, 결제 토큰을 PR 설명이나 댓글에 복사하지 않습니다.
- 리뷰 댓글 수보다 위험도와 근거를 우선합니다.
